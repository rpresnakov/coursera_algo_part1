    In the best case, the number of key compares to binary search for a key in a sorted array of N distinct keys is ~ lg N.
    		 Correct 0.20 It is 1, if the key is the middle element in the sorted array. 
    No compare-based algorithm can construct a BST from a sequence of N distinct keys in fewer than lg (N!) ~ N lg N key compares in the worst case.
    		 Correct 0.20 If it could, it would violate the ~ N lg N sorting lower bound because an inorder traversal (which takes linear time) gives the keys in sorted order.
